# Deploying EDII-TN LMS (single Render service)

One Render web service serves **both** the React frontend and the Express API.
Database is MongoDB Atlas.

```
Browser ──> Render service (Express)
              ├── /api/*        → API
              └── everything else → React app (client/dist)
Express ──> MongoDB Atlas
```

> ⚠️ Video note: Render's **free** tier has an ephemeral disk and sleeps after
> ~15 min idle, so uploaded videos in `/uploads` won't persist there. Everything
> else works. For persistent video, use a paid Render disk or Bunny Stream later.

---

## 1. Push to GitHub

```bash
cd LMS-EDII
git remote add origin https://github.com/Noburo-Business-Services-LLP/Edii-TN.git
git branch -M main
git push -u origin main
```
`.env` and `node_modules/` are gitignored — no secrets are committed.

## 2. Create the Render service

Dashboard → **New → Blueprint** → pick the `Edii-TN` repo (it reads `render.yaml`),
**or** New → Web Service with:
- Root Directory: *(repo root — leave blank)*
- Build: `cd client && npm install --include=dev && npm run build && cd ../server && npm install`
- Start: `cd server && npm start`
- Health Check Path: `/api/health`

Then set the one secret in **Environment**:
| Key | Value |
|-----|-------|
| `MONGO_URI` | your Atlas connection string (with `/edii-lms` db name) |

`JWT_ACCESS_SECRET` / `JWT_REFRESH_SECRET` are auto-generated by the Blueprint.
No `CLIENT_URL` needed — frontend and API share one origin.

## 3. Seed the database (once)

From your machine (points at the same Atlas DB):
```bash
cd server && npm run seed
```
Admin login: `admin@edii-tn.org` / `Admin@123`.

## 4. Open the app

Your Render URL (e.g. `https://edii-tn-lms.onrender.com`) serves everything —
the login page, catalog, player, and admin. Done.

---

### How it works
`VITE_API_URL` is left empty, so the built frontend calls `/api` on its own origin,
which the same Express server handles. `server/src/app.js` serves `client/dist`
with an SPA fallback for client-side routes.
